d# 解决应用配置缺失问题

## 问题

错误信息：`App configuration not found: plm_assistant`

## 原因

数据库中没有 `plm_assistant` 应用的配置记录。

## 解决方案

### 方案 1：通过 Web UI 创建应用配置

1. 打开 [http://localhost:8000](http://localhost:8000)
2. 点击 "Apps" 或 "Applications" 菜单
3. 点击 "New App" 或 "Create App" 按钮
4. 填写应用信息：
   - App Key: `plm_assistant`
   - Intent IDs: 选择需要的意图分类
   - Confidence Threshold: `0.7`（默认值）
   - Enable Cache: 启用
5. 保存配置

### 方案 2：通过 API 创建应用配置

使用 Admin API 创建应用配置：

```bash
curl -X POST http://localhost:8000/api/v1/admin/apps \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <admin_api_key>" \
  -d '{
    "app_key": "plm_assistant",
    "intent_ids": [1, 2, 3],
    "confidence_threshold": 0.7,
    "enable_cache": true
  }'
```

### 方案 3：直接插入数据库记录

如果数据库支持，可以直接插入一条应用配置记录：

```sql
INSERT INTO app_intents (app_key, intent_ids, confidence_threshold, enable_cache, created_at, updated_at)
VALUES ('plm_assistant', '[1, 2, 3]', 0.7, true, NOW(), NOW());
```

## 注意事项

1. **Intent IDs** 需要引用已存在的 `intent_categories` 表记录
2. **API Key** 用于标识应用，需要在请求中提供
3. **Confidence Threshold** 意图识别的置信度阈值，范围 0.0-1.0
4. **Enable Cache** 是否启用结果缓存，提升性能

## 验证

创建配置后，可以使用以下 API 验证：

```bash
curl http://localhost:8000/api/v1/admin/apps
```

应该能看到 `plm_assistant` 应用的配置。
